# Executive Summary - WellPlace (MVP)

## 1. Problem Statement

- La busqueda de espacios de bienestar locales es fragmentada y poco confiable.
- La informacion de horarios, cupos y precios suele estar desactualizada.
- Los usuarios pierden tiempo y enfrentan experiencias negativas al llegar.
- Los negocios no tienen presencia digital clara ni herramientas de reserva y cobro.

## 2. Solution Overview (MVP)

- Busqueda y comparacion de negocios con informacion verificada.
- Reservas obligatorias con seleccion de fecha, horario y cantidad de personas.
- Bloqueo de cupos en tiempo real para evitar sobreventa.
- Pago via QR local con confirmacion manual o automatica.
- Perfiles de negocio con fotos, precios, reglas y ratings basicos.

## 3. Success Metrics (KPIs)

- Conversion busqueda a reserva: >= 12%.
- Tasa de pago completado (pending_payment -> confirmed): >= 70%.
- Tiempo p95 de confirmacion de pago: <= 5 min en piloto.
- Negocios activos con disponibilidad actualizada semanalmente: >= 80%.
- Reservas completadas por usuario activo mensual: >= 1.5.

## 4. Target Users

- Usuario urbano recurrente: busca opciones cercanas y quiere reservar sin friccion.
- Usuario planificador familiar: necesita asegurar cupo y precio antes de salir.
- Dueno de negocio local: requiere visibilidad, reservas y cobro simple.
